# 콘마(Construction Master)

30억 미만 소규모 공사를 수행하는 중소 시공사를 위한 **AI-CM 설계도서 1차 검토 보조 서비스**입니다.

이 서비스는 AI가 도면, 시방서, 내역서를 완벽하게 판정하는 도구가 아닙니다.  
대신 시공사가 먼저 확인해야 할 위험 항목을 빠르게 찾고, 발주처·설계자·감리단에 확인할 내용을 정리하도록 돕습니다.

## 서비스 주소

- 웹 화면: https://ai-cm-review.vercel.app
- 서버 상태 확인: https://ai-cm-production.up.railway.app/api/health

## 누구를 위한 서비스인가요?

- 30억 미만 소규모 공사를 수행하는 중소 시공사
- 종합건설사와 전문건설사
- 설계도서 검토 인력이 부족해 우선 확인 항목을 빠르게 추리고 싶은 팀

## 무엇을 해주나요?

서비스는 업로드한 문서를 읽고, 아래 4가지 관점으로 검토 후보를 정리합니다.

1. **불일치·누락**
   - 도면이나 시방서에는 있는데 내역서에는 보이지 않는 항목을 찾습니다.

2. **RFI 후보**
   - 발주처, 설계자, 감리단에 공식 질의가 필요해 보이는 항목을 정리합니다.

3. **설계변경 검토**
   - 도면/시방서 기준과 내역 기준이 달라 계약 변경 검토가 필요할 수 있는 항목을 찾습니다.

4. **공사비 영향**
   - 내역 누락, 시험비 누락, 물량 차이, 신규 단가 가능성처럼 공사비에 영향을 줄 수 있는 항목을 표시합니다.

## 사용 흐름

1. **프로젝트 생성**
   - 공사별로 프로젝트를 만듭니다.
   - 프로젝트마다 파일과 검토 결과가 따로 저장됩니다.

2. **문서 업로드**
   - 도면, 시방서, 내역서를 업로드합니다.
   - 업로드한 파일은 서버 저장소에 보관됩니다.
   - 잘못 올린 파일은 화면에서 삭제할 수 있습니다.

3. **문서 추출 확인**
   - 서버가 문서에서 읽어낸 텍스트를 확인합니다.
   - 큰 PDF나 OCR이 필요한 문서는 시간이 더 걸릴 수 있습니다.

4. **AI 검토 실행**
   - 문서 추출 결과와 규칙 기반 후보를 바탕으로 AI가 1차 검토 항목을 정리합니다.

5. **결과 확인**
   - 4개 카테고리별 건수를 확인합니다.
   - 카테고리를 선택하면 해당 항목만 볼 수 있습니다.
   - 항목을 누르면 상세 내용, 관련 문서, 권장 조치, RFI 초안을 확인할 수 있습니다.

## 현재 직접 분석 가능한 문서

- 도면: PDF, 이미지 파일 `.jpg`, `.jpeg`, `.png`, `.webp`
- 시방서: PDF, `.hwpx`
- 내역서: Excel `.xlsx`, `.xlsm`

구형 `.hwp` 파일은 현재 직접 읽지 않습니다.  
한글 프로그램에서 `.hwpx` 또는 PDF로 저장한 뒤 업로드하는 방식이 안전합니다.

구형 Excel `.xls` 파일도 현재 직접 읽지 않습니다.  
Excel에서 `.xlsx`로 다시 저장한 뒤 업로드해 주세요.

## 내역서와 문서는 어떻게 읽나요?

- Excel 내역서는 여러 시트를 모두 확인합니다.
- 공종, 품명, 규격, 단위, 수량, 단가, 금액, 비고처럼 보이는 항목을 추출합니다.
- 표 구조를 완벽히 판단하지 못해도 각 시트의 셀 텍스트를 우선 읽습니다.
- 일반 PDF는 PDF 안에 들어 있는 텍스트를 먼저 읽습니다.
- 스캔 PDF나 도면 이미지는 OCR로 글자를 읽습니다.

## 어떤 공종을 확인하나요?

현재는 다음처럼 넓은 공종 키워드를 기준으로 후보를 찾습니다.

- 가설, 철거, 토공, 기초
- 콘크리트, 철근, 거푸집, 철골
- 조적, 미장, 방수, 단열, 지붕
- 석공, 타일, 도장, 수장, 바닥
- 창호, 유리, 금속, 목공, 가구
- 조경, 토목, 전기, 조명, 통신
- 소방, 기계설비, 냉난방, 가스, 승강기, 안전

예를 들어 도면/시방서에는 `전기`, `조명`, `방수`가 있는데 내역서에서 관련 항목이 보이지 않으면 누락 후보로 표시합니다.

## 결과 화면은 어떻게 구성되어 있나요?

- 대시보드: 프로젝트 상태와 검토 진행 상황 확인
- 프로젝트 관리: 공사별 프로젝트 생성 및 선택
- 문서 업로드: 도면, 시방서, 내역서 업로드와 삭제
- 추출 결과: 서버가 문서에서 읽은 내용 확인
- 검토 결과: 카테고리별 검토 결과 확인

검토 결과 화면은 항목이 많아져도 화면이 계속 길어지지 않도록 구성했습니다.  
카테고리 버튼을 선택하면 해당 카테고리의 항목만 아래에 출력됩니다.

## 서비스 구성

```text
사용자
  ↓
웹 화면 Vercel
  ↓
서버 Railway
  ↓
데이터베이스 PostgreSQL
  ↓
파일 저장소 Cloudflare R2
  ↓
OpenAI API
```

쉽게 말하면:

- **Vercel**: 사용자가 보는 웹사이트
- **Railway**: 업로드, 분석, AI 검토를 처리하는 서버
- **PostgreSQL**: 프로젝트와 검토 결과를 저장하는 데이터베이스
- **Cloudflare R2**: 업로드한 도면, 시방서, 내역서를 저장하는 파일 창고
- **OpenAI API**: OCR과 AI 검토에 사용

## AI가 판단할 때 쓰는 기준

AI는 혼자 추측해서 결론을 내리지 않도록 설정되어 있습니다.

현재 AI에게 전달되는 기준은 다음과 같습니다.

- 확정 판단을 하지 말 것
- 1차 검토 후보로만 분류할 것
- 근거가 부족하면 “확인 필요”로 표현할 것
- 결과는 아래 4개 카테고리 중 하나로 정리할 것
  - 불일치·누락
  - RFI 후보
  - 설계변경 검토
  - 공사비 영향

## 중요한 한계

이 서비스는 설계자나 감리자의 최종 판단을 대신하지 않습니다.

현재 한계는 다음과 같습니다.

- DWG 같은 CAD 원본의 선, 치수, 물량을 직접 계산하지는 않습니다.
- OCR은 이미지 품질이 낮으면 틀릴 수 있습니다.
- 도면 번호, 상세 위치, 시방 조항을 완벽히 연결하지는 못합니다.
- 내역서 양식이 복잡하면 일부 항목을 잘못 읽을 수 있습니다.
- 실제 계약 변경이나 공사비 청구 전에는 반드시 전문가 검토가 필요합니다.

## 앞으로 추가하면 좋은 기능

- 검토 결과 Excel/PDF 다운로드
- RFI 문서 자동 생성
- OCR 신뢰도 낮은 항목 표시
- 도면 번호와 시방 조항 자동 연결
- 사용자 로그인과 회사별 권한 관리
- 프로젝트별 협업자 초대

## 개발 정보

주요 파일은 다음과 같습니다.

```text
index.html        웹 화면 구조
styles.css        화면 디자인
script.js         화면 동작
server/src        서버 기능
```

서버 실행:

```bash
npm install
npm start
```

검사:

```bash
npm run check
```

환경변수는 GitHub에 올리지 않고 Railway에 등록합니다.

필요한 주요 값:

```text
DATABASE_URL
OPENAI_API_KEY
OPENAI_MODEL
OPENAI_OCR_MODEL
OCR_MAX_BYTES
UPLOAD_MAX_BYTES
R2_ACCOUNT_ID
R2_ACCESS_KEY_ID
R2_SECRET_ACCESS_KEY
R2_BUCKET_NAME
R2_PUBLIC_URL
CORS_ORIGIN
```
